# 13 · COMPONENTE: CONTATO

---

## Função narrativa

O Contato é o Capítulo V — o encerramento.

Após percorrer todo o edifício (Hero, Manifesto, About,
Projetos), o visitante chega à última página. Aqui não há
mais argumentos. Não há mais evidências. Há apenas
coordenadas para quem quiser continuar a conversa fora
do portfolio.

---

## Estrutura

O Contato ocupa o Espaço Arquivístico: funcional, direto,
sem atmosfera narrativa adicional.

**Cabeçalho:**
`[CONTATO]` ou `§ CONTATO` em grotesk bold. Sem ornamentos.

**Bloco de coordenadas:**
Formato de ficha técnica, similar ao About:

```
EMAIL            endereço@dominio.com
GITHUB           github.com/usuario
LINKEDIN         linkedin.com/in/usuario
LOCALIZAÇÃO      Cidade, País
```

Rótulos: monospace, azul cobalto, tamanho reduzido.
Valores: monospace, preto tinta.
Cada linha é uma unidade de contato. O símbolo `→` ou uma
marca de cota pode preceder cada linha como indicador de
"este é um canal".

**Texto de encerramento (opcional):**
Uma linha em serif italic, alinhada à direita, no rodapé da
seção. Ex: "A engenharia é a consequência material da
filosofia." Não é call to action. É ponto final.

---

## Sensação

O visitante deve sentir que chegou ao fim de um livro.

A última página de uma publicação não pede nada. Ela informa
onde o autor pode ser encontrado, lista os créditos, encerra
a experiência. O Contato tem a mesma função: é o colofão do
portfolio.

O tom não é "entre em contato comigo!" — é "se precisar,
aqui estão as coordenadas."

---

## Interação com links

Links de contato não são azuis sublinhados.

**Estado padrão:** texto monospace preto tinta, com uma
marca de referência cruzada ao lado (símbolo editorial de
"ver também" ou seta de cota apontando para fora).

**Hover:** interferência breve (glitch) no texto do link.
A marca de referência cruzada se destaca — como se o
documento dissesse "sim, este caminho leva para fora".

**Clique:** disjuntor (snap) — o visitante é levado ao
destino. Sem transição. Sem animação de saída. O portfolio
não retém o visitante.

---

## O que nunca fazer no Contato

- Nunca incluir formulário de contato (o portfolio não é
  um serviço — não coleta dados, não processa inputs).
- Nunca usar ícones de redes sociais (GitHub, LinkedIn).
  Use texto monospace — o nome do perfil é suficiente.
- Nunca usar call to action ("Vamos conversar!", "Entre
  em contato!", "Contrate-me!").
- Nunca usar tom persuasivo ou comercial.
- Nunca incluir foto do autor (não pertence ao
  encerramento — pertence ao About, se pertence).
- Nunca listar todas as redes sociais possíveis — apenas
  os canais onde o engenheiro está ativo e responde.

---

## Comportamento responsivo

**Amplo (desktop):** bloco de coordenadas em duas colunas
(rótulo + valor). Amplo respiro ao redor.

**Médio (tablet):** mantém o formato de duas colunas.

**Estreito (mobile):** os rótulos e valores podem empilhar
verticalmente se necessário, mas preferencialmente mantêm
o formato horizontal — é enxuto o suficiente para caber.

---

## Atmosfera material

Concreto polido. Sem textura intensa — o encerramento é
limpo. Os rótulos podem ter textura de carimbo (tinta
levemente irregular). Apenas uma cor de acento (azul
cobalto) além do preto tinta. O mínimo necessário para
ser funcional.
