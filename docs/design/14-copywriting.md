# 14 · COPYWRITING

---

## Princípio fundamental

Este projeto possui três vozes. Nenhuma delas é a voz de
um site convencional.

A voz errada destrói a atmosfera mais rápido do que a cor
errada. Um portfolio visualmente brutalista que diz "Olá,
eu sou desenvolvedor apaixonado por tecnologia" é uma
contradição que o visitante detecta em segundos.

---

## Registro 1 · Técnico-Poético

**Onde aparece:** Manifesto, Hero, declarações de princípio,
leads de seção, títulos de projeto.

**Tom:**
Declarativo. Preciso. Sem adjetivos de entusiasmo. Cada frase
é uma proposição — pode ser verdadeira ou falsa, mas nunca
vaga.

**Estrutura:**
Frases curtas. Parágrafos de 2 a 4 frases. Ritmo de prosa
técnica — como um paper de ciência da computação escrito
por alguém que leu Sêneca.

**Exemplos positivos:**
- "Engenharia é a consequência material da filosofia."
- "Cada sistema que construo é uma tese verificável."
- "A máquina obedece. O engenheiro decide."
- "Estrutura não é imposição. Estrutura é linguagem."

**Exemplos negativos:**
- "Sou apaixonado por tecnologia e inovação."
- "Transformo ideias complexas em soluções elegantes."
- "Acredito no poder do código para mudar o mundo."
- "Minha missão é criar produtos que encantam usuários."

**Padrões a evitar:**
- "Apaixonado por..." — o portfolio é sobre rigor, não paixão.
- "Transformar/impactar/mudar o mundo" — grandiosidade vazia.
- "Soluções inovadoras/criativas/elegantes" — adjetivo sem
  lastro.
- "UX/UI/experiência do usuário" — este portfolio não é sobre
  design de interfaces. É sobre engenharia.
- Qualquer frase que poderia estar no LinkedIn.

---

## Registro 2 · Arquivístico

**Onde aparece:** Metadados de projeto, legendas, coordenadas
de contato, campos de About, Footer.

**Tom:**
Funcional. Neutro. Sem personalidade expressiva. Como um
bibliotecário catalogando — a precisão é a única virtude.

**Estrutura:**
Campos rotulados. Formato consistente. Nenhuma variação
estilística entre entradas do mesmo tipo.

**Exemplos positivos:**
- `PROJETO 07 · SISTEMA DISTRIBUÍDO · 2024`
- `go · postgres · redis · grpc`
- `FIG. 03 — DIAGRAMA DE ARQUITETURA — 2025 — PROJETO XYZ`
- `NOME — SOBRENOME  ·  FORMAÇÃO, INSTITUIÇÃO, ANO`

**Exemplos negativos:**
- "Um projeto incrível que desenvolvi usando Go e Postgres"
- "Essa imagem mostra a arquitetura do sistema"
- "Me formei em Ciência da Computação pela universidade X"
- Qualquer texto que inclua opinião, entusiasmo ou narrativa.

**Padrões:**
- Use `·` (middle dot) como separador, não vírgula ou pipe.
- Monospace SEMPRE. Se o metadado está em serif, está errado.
- Azul cobalto para rótulos, preto tinta para valores.
- Datas no formato conciso: `2024`, `2024–2025`, `MM/AAAA`.
- Nunca usar descrições longas como metadado.

## Registro 3 · Funcional

**Onde aparece:** Navegação, labels, instruções implícitas,
textos de acessibilidade, elementos de interface.

**Tom:**
Direto. Sem verbo quando possível. Sem adjetivo. Como
sinalização de edifício: "SAÍDA", não "Por favor, dirija-se
até a saída."

**Exemplos positivos:**
- `MANIFESTO` (navegação)
- `PROJETOS`
- `CONTATO`
- `N PROJETOS REGISTRADOS`
- `ATUALIZADO EM MM/AAAA`

**Exemplos negativos:**
- "Veja meus projetos" (pedido, não rótulo).
- "Clique aqui para saber mais" (instrução redundante).
- "Explore meu trabalho" (convite, não sinalização).
- Qualquer texto que use verbos no imperativo para guiar o
  visitante.

**Padrões:**
- Navegação: substantivo. Não convida — informa onde você está
  ou para onde pode ir.
- Labels: monospace. Sempre.
- Instruções implícitas: a interface deve ser auto-evidente.
  Se precisa de texto explicativo, o design falhou.

## Nomes de projetos

Projetos possuem nomes, não títulos.

Um nome é uma designação — como o nome de uma peça de
engenharia. Um título é uma manchete.

**Exemplos positivos:**
- `SISTEMA DE ROTEAMENTO DISTRIBUÍDO`
- `COMPILADOR SUBSET C`
- `ENGINE DE BUSCA FULL-TEXT`
- `MONITOR DE INTEGRIDADE DE DADOS`

**Exemplos negativos:**
- "RouteMaster: O Roteador Inteligente"
- "C-Simp: compilando C de forma simples"
- "Searchify: Busca poderosa para sua aplicação"
- Nomes com sufixo "-ify", "-master", "-hub", "-lab".
- Nomes com pontuação decorativa (dois pontos, travessão).

O nome do projeto é uma etiqueta de arquivo, não uma marca.
Se o visitante lê o nome e pensa "isto parece um startup",
reformule.

## Texto do Hero

O Hero possui apenas dois elementos textuais:

1. O nome do portfolio: `CONCRETO, MENTE E TRÓPICOS`
2. O descritor: uma linha que define o que é este lugar.

**Opções de descritor (escolha uma):**
- `engenharia de software`
- `caderno técnico de projetos`
- `sistemas · estrutura · código`
- `arquivo de engenharia`
- `publicação técnica`

**O que nunca usar como descritor:**
- "Portfolio de Desenvolvedor" (redundante).
- "Full Stack Developer" (jargão de recrutamento).
- "Building the future, one line at a time" (clichê).
- Qualquer frase com verbo.

---

## Texto do Manifesto

O Manifesto é o texto mais autoral do portfolio. Deve ser
escrito pelo engenheiro — mas seguindo estas restrições:

- Máximo de 300 palavras.
- Cada parágrafo defende uma ideia, não descreve uma
  experiência.
- Nenhuma referência a tecnologias específicas (o Manifesto
  é sobre princípios, não sobre ferramentas).
- Nenhuma data, nenhum nome de empresa, nenhum cargo.
- Se possível, incluir uma referência a pensamento estoico
  ou engenharia como disciplina — sem soar pretensioso.

**Exemplo de estrutura:**
1. O que é engenharia de software (sua definição).
2. Por que a filosofia importa para o engenheiro.
3. Como você aplica isso na prática (sem exemplos específicos).
4. O que o visitante encontrará neste portfolio.

---

## Texto do About

O About é factual. Zero adjetivos opinativos.

**Campos obrigatórios:**
- Nome completo.
- Formação (curso, instituição, ano).
- Área de atuação (uma frase).
- Localização (cidade, país).
- Linguagens (lista).
- Áreas de interesse em engenharia (lista).

**Exemplo positivo:**

```
NOME             [Nome completo]
FORMAÇÃO         [Curso], [Instituição], [Ano]
ÁREA             [ex: Sistemas distribuídos e engenharia de dados]
LOCALIZAÇÃO      [Cidade], [País]
LINGUAGENS       go · python · c · rust · typescript
INTERESSE        compiladores · sistemas operacionais · protocolos
```

## Exemplos positivos e negativos por seção

| Seção | Certo | Errado |
|---|---|---|
| Hero | `engenharia de software` | `Olá, eu sou um desenvolvedor full stack` |
| Manifesto | "Engenharia é a consequência material da filosofia." | "Sou apaixonado por tecnologia desde criança." |
| About | `FORMAÇÃO  Ciência da Computação, USP, 2020` | "Formado em Ciência da Computação pela melhor universidade do Brasil." |

## O que nunca escrever em lugar nenhum

**Lista de palavras e expressões banidas:**

- Apaixonado
- Inovador
- Criativo
- Elegante (para código ou solução)
- Transformador
- Impacto (como verbo ou substantivo vazio)
- "Mudar o mundo"
- "Fazer a diferença"
- "Resolver problemas complexos" (sem especificar quais)
- "Tecnologias de ponta"
- "Clean code"
- "Boas práticas"
- "Full stack"
- "Ninja", "rockstar", "guru", "wizard"
- Qualquer emoji
- Qualquer exclamação
- "Olá!", "Bem-vindo!", "Seja bem-vindo!"

A regra é simples: se a frase poderia estar em um perfil
do LinkedIn sem chamar atenção, não pertence a este portfolio.
